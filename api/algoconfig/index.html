    device: str = DEFAULT_DEVICE pydantic-field

    Device to use for model inference. best will use the best available device.

    memory_strategy: str | None = DEFAULT_MEMORY_STRATEGY pydantic-field

    Memory strategy to use for model inference. high will use more memory, low will use less. Utilizing more memory will improve runtime performance.

    tqdm_enabled: bool = DEFAULT_TQDM_ENABLED pydantic-field

    Whether to enable tqdm progress bars.

    n_workers_for_norm_param_estimation: int = DEFAULT_N_WORKERS_FOR_NORM_PARAM_ESTIMATION pydantic-field

    Number of workers for norm parameter estimation from the baseline. Utilizing more workers will improve runtime performance and utilize more memory. Does not work with model compilation or MPS/GPU devices.

    batch_size_for_norm_param_estimation: int = DEFAULT_BATCH_SIZE_FOR_NORM_PARAM_ESTIMATION pydantic-field

    Batch size for norm parameter estimation from the baseline. Utilizing a larger batch size will improve runtime performance and utilize more memory.

    stride_for_norm_param_estimation: int = DEFAULT_STRIDE_FOR_NORM_PARAM_ESTIMATION pydantic-field

    Stride for norm parameter estimation from the baseline. Utilizing a larger stride will improve metric accuracy and utilize more memory.Memory usage scales inverse quadratically with stride. That is, If stride=16 consumes N bytes of memory, then stride=4 consumes 16N bytes of memory.

    apply_logit_to_inputs: bool = DEFAULT_APPLY_LOGIT_TO_INPUTS pydantic-field

    Whether to apply logit transform to the input data.

    n_workers_for_despeckling: int = DEFAULT_N_WORKERS_FOR_DESPECKLING pydantic-field

    Number of workers for despeckling. Utilizing more workers will improve runtime performance and utilize more memory.

    lookback_strategy: str = DEFAULT_LOOKBACK_STRATEGY pydantic-field

    Lookback strategy to use for data curation of the baseline. multi_window will use a multi-window lookback strategy and is default for OEPRA DIST-S1, immediate_lookback will use an immediate lookback strategy using acquisitions preceding the post-date. immediate_lookback is not supported yet.

    post_date_buffer_days: int = DEFAULT_POST_DATE_BUFFER_DAYS pydantic-field

    Buffer days around post-date for data collection to create acqusition image to compare baseline to.

    model_compilation: bool = DEFAULT_MODEL_COMPILATION pydantic-field

    Whether to compile the model for CPU or GPU. False, use the model as is. True, load the model and compile for CPU or GPU optimizations.

    max_pre_imgs_per_burst_mw: tuple[int, ...] | None = DEFAULT_MAX_PRE_IMGS_PER_BURST_MW pydantic-field

    Max number of pre-images per burst within each window

    delta_lookback_days_mw: tuple[int, ...] | None = DEFAULT_DELTA_LOOKBACK_DAYS_MW pydantic-field

    Delta lookback days for each window relative to post-image acquisition date

    low_confidence_alert_threshold: float = DEFAULT_LOW_CONFIDENCE_ALERT_THRESHOLD pydantic-field

    Low confidence alert threshold for detecting disturbance between baseline and post-image.

    high_confidence_alert_threshold: float = DEFAULT_HIGH_CONFIDENCE_ALERT_THRESHOLD pydantic-field

    High confidence alert threshold for detecting disturbance between baseline and post-image.

    no_day_limit: int = DEFAULT_NO_DAY_LIMIT pydantic-field

    Number of days to limit confirmation process logic to. Confirmation must occur within first observance of disturbance and no_day_limit days after first disturbance.

    exclude_consecutive_no_dist: int = DEFAULT_EXCLUDE_CONSECUTIVE_NO_DIST pydantic-field

    Boolean activation of consecutive no disturbance tracking during confirmation. True will apply this logic: after 2 no disturbances within product sequence, the disturbance must finish or be reset. False will not apply this logic.

    percent_reset_thresh: int = DEFAULT_PERCENT_RESET_THRESH pydantic-field

    Precentage number threshold to reset disturbance. Values below percent_reset_thresh will reset disturbance.

    no_count_reset_thresh: int = DEFAULT_NO_COUNT_RESET_THRESH pydantic-field

    If the number of non-disturbed observations prevnocount is above nocount_reset_thresh disturbance will reset.

    max_obs_num_year: int = DEFAULT_MAX_OBS_NUM_YEAR pydantic-field

    Max observation number per year. If observations exceeds this number, then the confirmation must conclude and be reset.

    confidence_upper_lim: int = DEFAULT_CONFIDENCE_UPPER_LIM pydantic-field

    Confidence upper limit for confirmation. Confidence is an accumulation of the metric over time.

    confirmation_confidence_threshold: float = DEFAULT_CONFIRMATION_CONFIDENCE_THRESHOLD pydantic-field

    This is the threshold for the confirmation process to determine if a disturbance is confirmed.

    metric_value_upper_lim: float = DEFAULT_METRIC_VALUE_UPPER_LIM pydantic-field

    Metric upper limit set during confirmation

    model_source: str = DEFAULT_MODEL_SOURCE pydantic-field

    Model source. If external, use externally supplied paths for weights and config. Otherwise, use distmetrics.model_load.ALLOWED_MODELS for other models.

    model_cfg_path: Path | str | None = DEFAULT_MODEL_CFG_PATH pydantic-field

    Path to model config file. If external, use externally supplied path. Otherwise, use distmetrics.model_load.ALLOWED_MODELS for other models.

    model_wts_path: Path | str | None = DEFAULT_MODEL_WTS_PATH pydantic-field

    Path to model weights file. If external, use externally supplied path. Otherwise, use distmetrics.model_load.ALLOWED_MODELS for other models.

    apply_despeckling: bool = DEFAULT_APPLY_DESPECKLING pydantic-field

    Whether to apply despeckling to the input data.

    interpolation_method: str = DEFAULT_INTERPOLATION_METHOD pydantic-field

    Interpolation method to use for despeckling. nearest will use nearest neighbor interpolation, bilinear will use bilinear interpolation, and none will not apply despeckling.

    model_dtype: str = DEFAULT_MODEL_DTYPE pydantic-field

    Data type for model inference. Note: bfloat16 is only supported on GPU devices.

    use_date_encoding: bool = DEFAULT_USE_DATE_ENCODING pydantic-field

    Whether to use acquisition date encoding in model application (currently not supported)

    n_anniversaries_for_mw: int = DEFAULT_N_ANNIVERSARIES_FOR_MW pydantic-field

    Number of anniversaries to use for multi-window
